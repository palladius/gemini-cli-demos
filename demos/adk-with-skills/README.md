This demo showcases how `gemini-cli` is able to:

* Create code to generate working ADK code. Python v1.25.0+ apparently.
* Use Skills as described in the https://adk.dev/skills/
  * Use Riccardo's `nano-banana-ricc` skill to generate an image of Riccardo using ADK.
* Push it to [GEAP](https://docs.cloud.google.com/gemini-enterprise-agent-platform) 

## How to use this demo

1. Show how it's easy to generate ADK code with ADK skill.

## REQUIREMENTS

* Python 3.10+
* `uv` for dependency management
* `google-adk` package
* Access to Gemini API keys.

**Note**. Currently leveraging a skill 100% doesn't work -> https://github.com/google/adk-python/issues/5524 .

## Features

*   **Character Consistency**: Uses the `nano-banana-ricc` skill with reference images to generate consistent images of Riccardo.
*   **Resilient Design**: Handles 503 (High Demand) and 429 (Quota) errors gracefully using an `ErrorHandlingPlugin`.
*   **GCS-Backed Assets**: Reference images are hosted on GCS to keep the deployment payload small and comply with Agent Engine limits.
*   **Artifact Attachment**: Generated images are saved as ADK artifacts, making them easily viewable in the GEAP Playground and local web UI.
*   **Observability**: Full OpenTelemetry instrumentation and message content capture enabled for debugging in GEAP.

## Deployment Permissions

Before deploying to GEAP (Agent Engine), ensure you have the necessary IAM roles. You can run the provided script to grant yourself the `roles/aiplatform.admin` role:

```bash
./bin/grant_permissions.sh
```

## Known Issues

*   The `SkillToolset` is not yet fully supported with the `BuiltInCodeExecutor`. See [docs/ADK-SkillToolset-BUG.md](./docs/ADK-SkillToolset-BUG.md) for technical details and status of [issue #5524](https://github.com/google/adk-python/issues/5524).
*   We've added an `ErrorHandlingPlugin` to handle 503/UNAVAILABLE errors gracefully without crashing the CLI.
*   The corp airlock issue was resolved by setting `UV_INDEX_URL="https://pypi.org/simple"`.
